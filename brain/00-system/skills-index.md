# Capability Registry

What this brain can *do* (executable skills), vs what it *knows* (typed nodes). Maintained by
`organize-vault` / `skillify`.

## Core operations
- `convert-note` - raw material -> typed atomic nodes
- `query-vault` - routing-first retrieval
- `organize-vault` - graduate notes, upgrade edges, find orphans/contradictions
- `vault-health` - confidence decay + staleness sweep (claim-types)
- `init-vault` - scaffold / repair
- `ingest` / `recall` - LightRAG Tier-2 bridge
- `import` - bring Google Drive / Notion exports into the brain (Tier-2 + curated; URL-rewriting)
- `start-project` / `close-project` - project lifecycle (playbook -> project -> tasks -> learnings)
- `new-brain` - clone a fresh isolated vault from the template
- `skillify` - compile a playbook into a new skill (the brain->skill loop)
